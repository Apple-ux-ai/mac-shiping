macOS / Windows 打包说明

项目栈
- Electron + React(Vite) 前端：frontend/
- Python 后端：backend/，PyInstaller 产出 backend/dist/api/
- electron-builder：electron-builder.yml

Windows
- 在 Windows 上执行：npm install，再 npm run build
- 产物：release/<version>/installer、release/<version>/portable
- 内置更新：updater.exe 由 build_backend.py 构建并复制到 dist/updater.exe 供 electron-builder 打包

macOS 便携 zip（未签名）
- 本地无 Mac 时：使用 GitHub Actions（见下文「GitHub Actions 云端构建」）
- 有 Mac 时本地构建：无法在 Windows 上产出可用的 .app
- 依赖：Node.js、Python 3、PyInstaller；系统需安装 ffmpeg / ffprobe（且在打包后的环境中仍在 PATH 中可用，或使用自备二进制并自行接入）
- 步骤：
  1. cd 到本目录（含 package.json）
  2. npm install
  3. npm run build:frontend
  4. python build_backend.py
  5. npx electron-builder --mac
- 产物：release/<version>/portable/视频格式转换大师-mac-<arch>-<version>.zip
- 解压后得到 .app；未签名时若被拦截，可在「隐私与安全性」中允许，或使用右键「打开」首次运行
- mac 上应用内「启动更新程序」已禁用（仅 Windows 使用 updater.exe）

可选资源
- build/icon.icns：Dock/窗口图标（无则使用默认）
- Windows 窗口图标：build/icon.ico

GitHub Actions 云端构建（推荐无 Mac 时使用）
- 工作流文件：.github/workflows/build-mac-portable.yml
- 触发：push 到 main 且变更命中 workflow 中 paths 所列目录；或在 Actions 里手动「Run workflow」
- 使用矩阵：macos-latest（arm64）与 macos-15-intel（x64），各生成一个 zip，在运行结果页「Artifacts」下载 mac-portable-arm64 / mac-portable-x64
- 私有仓库会消耗 Actions 分钟数；公开仓库使用标准运行器按 GitHub 规则计费/免费额度
- 若 Intel 任务因运行器不可用失败，可编辑 workflow 去掉 macos-15-intel 那一行，仅保留 arm64
- 构建进行途中请勿连续 push 同一分支并依赖「取消旧运行」：旧版 workflow 曾用 cancel-in-progress，会导致 electron-builder 报 “The operation was canceled”；当前已关闭该行为
- electron-builder：配置里使用 publish: null（不是 publish: never）；never 会被当成 provider 名并在 build/ 下查找 electron-publisher-never.js 导致报错；命令行用 --publish never 表示「不执行上传」
